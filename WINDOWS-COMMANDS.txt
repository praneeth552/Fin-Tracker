====================================
FINTRACKERAPP - WINDOWS QUICK COMMANDS
====================================

FOR TEAM MEMBERS WHO ALREADY HAVE THE REPO CLONED
--------------------------------------------------

STEP 1: UPDATE YOUR CODE
------------------------
cd FinTrackerApp
git pull origin main


STEP 2: START DOCKER DESKTOP
-----------------------------
- Click Start menu
- Search "Docker Desktop"
- Open it and wait for it to start completely
- Look for whale icon in system tray (bottom right)


STEP 3: START THE APP
---------------------
(in PowerShell or Command Prompt, inside FinTrackerApp folder)

docker-compose up -d


STEP 4: CHECK IF RUNNING
------------------------
docker-compose ps

You should see both "frontend" and "backend" with status "Up"


DAILY COMMANDS
--------------
Start everything:
docker-compose up -d

Stop everything:
docker-compose down

View logs:
docker-compose logs -f

Check status:
docker-compose ps

Build APK:
docker-compose exec frontend bash -c "cd android && ./gradlew assembleDebug"

Enter frontend container:
docker-compose exec frontend /bin/bash

Enter backend container:
docker-compose exec backend /bin/bash


INSTALLING APK ON YOUR PHONE
-----------------------------
1. Enable USB debugging on phone:
   - Settings → About Phone → Tap "Build Number" 7 times
   - Settings → Developer Options → Enable "USB Debugging"

2. Install ADB (choose one method):
   
   Method A - Chocolatey (easiest):
   choco install adb
   
   Method B - Download Platform Tools:
   Download from: https://developer.android.com/tools/releases/platform-tools
   Extract to C:\platform-tools
   
3. Add ADB to PATH (if not using Chocolatey):
   - Right-click "This PC" → Properties
   - Advanced System Settings → Environment Variables
   - Under "System Variables", find "Path" → Edit → New
   - Add: C:\Users\YourName\AppData\Local\Android\Sdk\platform-tools
   - Or: C:\platform-tools
   - Click OK, restart PowerShell

4. Connect phone via USB

5. Verify ADB works:
   adb devices
   
6. Install the APK:
   adb install android\app\build\outputs\apk\debug\app-debug.apk


TROUBLESHOOTING
---------------
Problem: "docker-compose not recognized"
Solution: Make sure Docker Desktop is running

Problem: "adb is not recognized"
Solution: 
  1. Make sure ADB is installed (see INSTALLING APK section)
  2. Add to PATH and restart PowerShell
  3. Test with: adb version

Problem: Port already in use
Solution: Stop other apps on port 8080/8081

Problem: Android SDK API 36 download fails
Solution: 
  Edit Dockerfile.frontend, change:
  "platforms;android-36" to "platforms;android-35"
  "build-tools;36.0.0" to "build-tools;35.0.0"
  Then rebuild: docker-compose build --no-cache frontend

Problem: Need to rebuild
Solution: 
  docker-compose down -v --rmi all
  docker-compose build --no-cache
  docker-compose up -d
